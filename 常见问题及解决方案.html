使用前，请先保证

  1. 你的Python环境是：Python36/37/38/39
  2. 你的杀毒软件没有删除libs目录下的dll文件
  3. 安装的微信版本正确

常见问题及解决方案

常见问题1：

  • 原因1：请检查你的微信版本是否是指定版本
  • 原因2：请检查你的杀毒软件是否把libs文件夹中的dll文件拦截了
  • 解决方案：安装指定版本的微信、把dll文件添加至你杀毒软件的白名单中

常见问题2：

  • 原因：这是因为你的Python环境不是36/37/38/39，或者没有安装依赖包。

  • 解决方案：

    • 使用对应版本的Python环境

    • 初次使用需要pip安装四个库：

      • pip install requests
      • pip install pycryptodomex
      • pip install psutil
      • pip install pywin32

    • 若还不行，请把其他pyd删除掉(你是Python36。就删除37、38、39的pyd文件，其他版本同理)

常见问题3：

  • 原因：如果遇到这种问题，请检查你的杀毒软件是否把libs目录下的dll文件误删了
  • 解决方案：关闭杀毒软件或者把WeChatApi.dll设置为白名单

常见问题4：

  • 导致原因1：你的Python环境是64位的，但是你正在使用32位的pyd文件，反之同理

  • 解决方案：源码中已提供32位跟64位的pyd文件，选择对应的pyd文件进行导包即可

  • 导致原因2：你可能误删了pyd文件，然后又把它从回收站恢复回来，导致编码错误

  • 解决方案：此时你只能重新pull一遍项目了

常见问题5

  • 我的Python环境没问题、用的也是对应版本的微信，为什么实例化的时候还是报错？
  • 类似这种错误： 《ImportError: DLL load failed: 找不到指定的模块》

  • 原因：包的版本太高也可能会导致实例化失败，主要是《pywin32》跟《pycryptodomex》这两个包

  • 解决方案：

    • 尝试降低或升高包的版本
    • 以下是我本机环境的包版本，供参考：
      • requests==2.23.0
      • pycryptodomex==3.9.4
      • psutil==5.7.0
      • pywin32==301

常见问题6：

  • 原因：

    • 有很多小伙伴不太了解pyd文件，在导包时，Pycharm报红，以为这是一个error

    • 于是他把文件名后缀改为.py

  • 解决方案：报红为正常现象，请不要随意改动文件后缀

  • 这里说一点：很多小伙伴觉得pyd文件名长的恶心，比如这样：【WeChatPYAPI.cp38-win_amd64.pyd】，其实中间那坨是可以删掉的，给你们的时候留着是为了你们好分辨版本，实际项目中可以改成下图这样：

常见问题7（一启动就崩溃）：

  • 原因：没有使用对应微信版本的包

  • 解决方案：

    • 如果你使用的是免费版
      • 安装免费版指定版本的微信
      • 使用免费版的包
    • 如果你使用的是专业版
      • 安装专业版指定版本的微信
      • 使用专业版的包

其他问题

  • 问题1：很多小伙伴会使用pyinstaller对自己的项目进行打包，但是打包后发现无法正常使用WeChatPYAPI

  • 答：如果你使用“-F”打包为单文件，那么记得把libs文件夹载入你的虚拟目录中，如果你打包为多文件，那么直接把libs文件夹拷贝至你的项目目录中

  • 问题2：为什么使用pyinstaller打包时，出现包不存在的问题

  • 答：因为pyinstaller打包时，会检测pyd中使用到的包，如果你使用pyinstaller打包，请在你的main文件头部添加以下代码：

  • import json, requests, win32con, win32api, psutil
from Cryptodome.Cipher import AES
try:
    import xml.etree.cElementTree as ET
except ImportError:
    import xml.etree.ElementTree as ET

  • 问题3：我可以自行指定libs目录路径吗？

  • 答：可以，如果你不想在项目中有libs文件夹，在实例化WeChatPYApi的时候，传入参数：libs_dir=你自定义目录的绝对路径。虽然目录可更改，但是！ WeChatApi.dll跟 inject_tool.exe两个文件必须在同一级目录下：

  • 问题4：如果你使用免费版本时出现了崩溃的问题，很不幸，这个问题在免费版没有解决掉。
